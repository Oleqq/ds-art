# Текущий статус проекта

Этот файл фиксирует финальный рабочий срез проекта без хронологии мелких правок. История этапов вынесена в `phases/`, а здесь остается только актуальное состояние.

## Релизный срез

DS Art Staff Cabinet находится в почти релизном состоянии:

- основные бизнес-сценарии собраны;
- база знаний, поиск, ACL и employee-onboarding работают в приложении;
- документация и deploy-процесс приведены к финальному формату;
- текущий фокус - production smoke после деплоя и реальная эксплуатационная настройка.

## Что уже готово

### Платформа

- Laravel `13` + Inertia + React + TypeScript;
- MySQL-миграции и сиды;
- layouts для `admin`, `employee`, `auth`, `settings`;
- локальные проверки через Pint, ESLint, TypeScript и PHPUnit.

### Сотрудники

- список сотрудников;
- карточка сотрудника;
- создание, редактирование, деактивация;
- фото профиля;
- файлы сотрудника;
- employee self-service профиль.

### База знаний

- дерево разделов и категорий;
- статьи и блочный редактор;
- обложки, иконки, изображения, видео и файлы;
- дублирование, перемещение, публикация;
- экспортные и вспомогательные сценарии вокруг статьи;
- представления для `admin` и `employee`.

### Доступ и поиск

- персональные права на разделы и статьи;
- backend-проверки доступа;
- `access_level` статьи;
- поиск по категориям, статьям, блокам и asset-данным;
- search-flow с учетом прав конкретного сотрудника.

### Безопасность и аудит

- роли `admin` и `employee`;
- только активные сотрудники допускаются в рабочий кабинет;
- аудит критичных действий;
- экран безопасности и смены пароля;
- onboarding без временного пароля.

### Почта и активация

- регистрация сотрудника через `/register`;
- отправка 6-значного кода на рабочую почту;
- установка собственного пароля;
- поддержка `SMTP_*` с приоритетом над `MAIL_*`.

### Deploy

- companion manual deploy-пакет обновляется отдельно в:
  - `C:\dev\ds-art-manual-deploy\ds-art-app`
  - `C:\dev\ds-art-manual-deploy\public_html`
- есть документация для ручного FTP-deploy;
- есть GitHub Actions workflow для автоматической заливки файлов.

## Что остается

### Перед production-эксплуатацией

- выполнить финальный smoke-проход по production после полного обновления версии;
- убедиться, что на проде выполнены миграции и настроены mail/storage-пути;
- проверить реальные права сотрудников на контент уже в production-базе.

### Не блокирует релиз, но может дорабатываться

- точечный UI-polish по живым замечаниям;
- дополнительная контентная шлифовка базы знаний;
- развитие редактора за пределами обязательного MVP.

### Осознанно отложено

- AI-агент по базе знаний;
- любые AI-надстройки поверх поиска и статей;
- внешние публичные API и отдельное мобильное приложение.

## Последние контрольные проверки

Для текущего релизного среза уже прогонялись:

- `php artisan test --filter=KnowledgeBasePagesTest --stop-on-failure`
- `php artisan test --filter=EmployeeAssetsTest --stop-on-failure`
- `php artisan test --filter=EmployeeActivationTest --stop-on-failure`
- `npm run types:check`
- `vendor/bin/pint --test ...`
- `npm run build`

Для production-сборки на этой машине использовался Herd Node `v25.9.0`, потому что системный Node `18.x` уже не подходит под текущий Vite.
