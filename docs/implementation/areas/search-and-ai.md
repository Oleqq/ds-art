# Область: Поиск и AI

## Цель

Дать пользователю быстрый доступ к нужному контенту и позже построить поверх этого AI-слой.

## Поиск в MVP

В MVP нужен рабочий поиск по БЗ.

Минимально:

- поиск по заголовкам статей
- поиск по содержимому статей
- поиск с учетом прав пользователя

## Технологический подход

Сначала:

- обычный поиск по MySQL

Потом, если это действительно нужно:

- `Scout`
- `Meilisearch`

## AI-агент

AI-агент не входит в первый обязательный этап разработки.

Он делается после того, как готовы:

- нормальная схема БД
- права доступа
- поиск
- чистые предметные сущности

## Что должен уметь AI-слой

- находить статьи
- отвечать по базе знаний
- ориентироваться в данных сотрудников, статей, файлов и других сущностей
- ускорять поиск информации внутри системы

## Главное правило для AI

AI не может иметь доступ шире, чем пользователь.

Каждый AI-запрос должен уважать ACL приложения.
