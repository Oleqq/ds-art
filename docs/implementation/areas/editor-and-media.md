# Область: Редактор и медиа

## Цель

Реализовать редактор статьи так, чтобы он покрывал сценарии прототипа и оставался поддерживаемым.

## Что есть в прототипе

В прототипе уже есть:

- заголовок статьи
- обложка
- эмодзи-иконка
- блоки контента
- текст
- списки
- цитаты
- код
- ссылки
- изображения
- таблицы
- файлы
- видео
- drag-and-drop блоков

## Что нужно реализовать

Все это относится к MVP.

Но реализация должна быть инженерно устойчивой.

## Подход

На старте:

- реализовать редактор под реальные сценарии прототипа
- не втаскивать тяжелый editor stack без необходимости

Если редакторная логика становится слишком хрупкой:

- переводим редактор на `Tiptap`

## Медиа

Нужно поддержать:

- обложки статей
- изображения внутри статьи
- видео по ссылке
- файловые блоки

## Важное правило

Нельзя хранить production-медиа как base64 внутри БД, если это не временная внутренняя форма.

Финальное хранение должно идти через файловое хранилище.
