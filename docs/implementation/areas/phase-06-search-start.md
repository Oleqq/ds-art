# Этап 06. Старт поиска по базе знаний

Дата: 22.04.2026

## Что сделано

- Локальная MySQL-БД поднята через `scripts/start-local-mysql.ps1`; `mysqld` слушает `127.0.0.1:3306`.
- Проверено состояние БД: `users` — 2, `knowledge_categories` — 15, `knowledge_articles` — 19, `knowledge_article_blocks` — 38, `knowledge_article_assets` — 0.
- Добавлен backend-поиск по базе знаний через отдельный сервис `KnowledgeBaseSearch`.
- Поиск идет не по фронтовой верстке, а по SQL-данным: разделы, статьи, `knowledge_article_blocks.plain_text/html`, `knowledge_article_assets`.
- Добавлены Inertia-страницы результатов поиска для `admin` и `employee`.
- В sidebar вместо заглушки теперь рабочий input поиска.

## Важное архитектурное решение

Поиск сразу строится от backend-данных и нормализованных блоков статьи. Это нужно, чтобы потом:

- корректно учитывать права доступа;
- не отдавать сотруднику скрытые или черновые материалы;
- использовать эти же данные для будущего AI-агента;
- проще переносить базу знаний на другой фронт или клиент.

## Правила доступа в текущей реализации

- `admin` ищет по всем разделам и статьям, включая черновики и скрытые материалы.
- `employee` ищет только по опубликованным статьям из видимых разделов.
