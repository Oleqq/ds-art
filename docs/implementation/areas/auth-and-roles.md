# Область: Вход, регистрация и роли

## Цель

Реализовать вход в систему и разграничение доступа так, чтобы это было безопасно и соответствовало корпоративному приложению.

## Что есть в прототипе

В HTML-прототипе пользователь попадает в систему по email и видит либо admin, либо employee часть.

## Как делаем в реальном приложении

В MVP нужна нормальная аутентификация.

## Регистрация

По умолчанию открытая публичная регистрация не нужна.

Предполагаем такой сценарий:

- сотрудника создает или активирует администратор
- пользователю назначается учетная запись
- пользователь входит в систему по логину и паролю

Если бизнес позже захочет self-registration, это отдельная задача, не базовый MVP.

## Базовые роли

- `admin`
- `employee`

## Базовые проверки

- пользователь существует
- пользователь активен
- сотрудник активен
- пользователь имеет право на нужный layout и экран

## Что должно работать

- вход
- выход
- защита маршрутов
- разделение интерфейсов admin/employee
- блокировка входа неактивным пользователям

## Особое правило

Права на экраны и права на бизнес-действия не смешиваются.

Нужно отдельно проверять:

- доступ к экрану
- доступ к данным
- доступ к действию
