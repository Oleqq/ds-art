# Область: Файлы и хранение

## Цель

Сделать работу с файлами безопасной и предсказуемой.

## Что входит

- фото сотрудника
- вложения сотрудника
- вложения статьи
- изображения и медиа статьи

## Как храним

Файлы должны идти через `Laravel Storage`.

В БД храним:

- путь
- оригинальное имя
- mime type
- размер
- тип владельца
- идентификатор владельца

## Что нельзя делать

- хранить все файлы как base64 в БД
- смешивать employee files и article files без явной модели
- отдавать файл без backend-проверки доступа

## Что нужно проверить

- кто загрузил файл
- к какой сущности файл привязан
- имеет ли пользователь право его видеть
- можно ли удалять или заменять этот файл
