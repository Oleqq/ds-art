# Область: Backend

## Цель

Собрать надежный Laravel-бэкенд без избыточной архитектурной сложности.

## Базовый подход

Проект строится как модульный монолит.

Ключевые правила:

- контроллеры тонкие
- валидация через Form Request
- прикладные сценарии выносятся в Actions/Services
- авторизация через Policies и Gates
- транзакции используются там, где меняются связанные сущности

## Что должен покрывать backend

- аутентификацию
- авторизацию
- роли и статусы
- сотрудников
- категории БЗ
- статьи
- блоки статьи
- файлы
- права доступа
- поиск

## Рекомендуемые доменные зоны

- `Auth`
- `Employees`
- `KnowledgeBase`
- `Access`
- `Files`
- `Audit`

## Что не нужно делать на старте

- отдельный API gateway
- отдельный backend-only repo
- микросервисы
- repository layer “потому что так принято”
- event sourcing
- CQRS без реальной боли

## Обязательные backend-проверки

- активен ли пользователь
- имеет ли роль на просмотр экрана
- имеет ли право на конкретный раздел БЗ
- имеет ли право на конкретную статью
- имеет ли право на редактирование или удаление
- имеет ли право на просмотр или загрузку файла
