# Область: Сотрудники

## Цель

Реализовать весь модуль сотрудников из HTML-прототипа.

## Что входит

- список сотрудников
- поиск
- фильтрация
- карточка сотрудника
- редактирование сотрудника
- статус active/inactive
- график работы
- заметки руководителя
- файлы сотрудника
- фото сотрудника

## Представления

Нужно два разных режима:

- admin view
- employee self view

## Важно

Сотрудник не должен видеть:

- приватные заметки руководителя
- административные управляющие действия
- данные, которые не относятся к self-service

## Минимальная модель

- сотрудник
- учетная запись
- статус
- роль
- график
- заметки
- вложения

## Основные действия

- создать сотрудника
- изменить сотрудника
- активировать/деактивировать сотрудника
- загрузить фото
- загрузить файл
- удалить файл
