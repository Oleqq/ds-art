# Область: Frontend

## Цель

Перенести поведение HTML-прототипа в рабочее приложение без отдельного SPA-репозитория.

## Базовый стек

- Inertia.js
- React
- TypeScript
- Vite

## Принцип

Фронтенд не живет отдельно от Laravel.

Laravel отвечает за:

- URL
- доступ
- отдачу страниц

React отвечает за:

- интерактивность
- формы
- локальное состояние
- сложные компоненты

## Обязательные части интерфейса

- admin layout
- employee layout
- навигация по сотрудникам
- навигация по БЗ
- модальные окна
- таблицы
- карточки
- редактор статьи
- drag-and-drop там, где это есть в прототипе

## Состояние

На старте используем:

- локальный component state
- Inertia props

Не добавляем сразу:

- Redux
- Zustand
- React Query

## Стиль переноса верстки

Нельзя просто “собрать заново похожий интерфейс”.

Нужно перенести:

- структуру экранов
- состав экранов
- основные сценарии
- ключевую интерактивность

## Что считается отклонением

Отклонение от MVP, если:

- убрана функция из прототипа
- упрощен экран так, что ломается пользовательский сценарий
- права и поведение интерфейса перестают соответствовать прототипу
