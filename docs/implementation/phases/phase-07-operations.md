## File Storage Strategy

Сейчас файлы хранятся на Laravel `public` disk:

- сотрудники: фото и файлы
- разделы базы знаний: обложки и иконки
- статьи: обложки, изображения, файлы, видео

Production-условие:

- [ ] `storage/app/public` должен быть доступен через `public/storage`.

Если SSH есть:

```bash
php artisan storage:link
```

Если SSH нет, symlink нужно создать через панель хостинга или согласовать с поддержкой.

- [ ] Если хостинг даст SSH, можно делать GitHub Actions deploy: build, upload по SSH/SFTP, затем `php artisan migrate --force`, `storage:link`, `optimize`.

## Redis, Очереди, Horizon, Pulse

Пока не подключаем:

- [ ] Redis
- [ ] Horizon
- [ ] Pulse
- [ ] queue workers

Причина: сейчас нет доказанной необходимости. В MVP нет тяжелых фоновых задач, email-рассылок, индексации внешнего поиска или высокой нагрузки.

Когда подключать:

- Redis — если file cache/session начнут тормозить или появится несколько серверов.
- Очереди — если появятся тяжелые задачи: обработка файлов, генерация превью, массовые уведомления, AI-анализ.
- Horizon — только когда реально появятся очереди на Redis.
- Pulse — если понадобится runtime-наблюдаемость на production.

## Soft Delete

Пока не включено глобально.

Решение:

- [ ] Если нужен, добавлять осознанно: миграции, модели, восстановление, UI “архива”, тесты.

Почему не добавляем автоматически: soft delete без интерфейса восстановления часто создает скрытую сложность и мусор в данных.
