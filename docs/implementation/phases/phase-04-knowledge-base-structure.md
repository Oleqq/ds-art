# Этап 04. Структура базы знаний

## Цель

Собрать структурный слой базы знаний: дерево разделов, главную страницу БЗ, экран категории, навигацию в sidebar и admin CRUD для разделов с паритетом по поведению и визуалу относительно прототипа.

## Что должно быть на выходе

- главная страница базы знаний
- дерево разделов в sidebar
- корневые разделы и подразделы
- экран категории
- breadcrumbs
- demo-структура из HTML-прототипа
- разный режим для `admin` и `employee`
- hover-controls и контекстные действия в дереве
- красивый confirm flow вместо нативных alert/confirm
- file-based обложка раздела

## Что реализовано

- [x] таблицы `knowledge_categories` и `knowledge_articles`
- [x] demo-seed дерева БЗ на основе HTML-прототипа
- [x] главная БЗ для `admin`
- [x] главная БЗ для `employee`
- [x] экран категории для `admin`
- [x] экран категории для `employee`
- [x] реальные данные БЗ в sidebar
- [x] CRUD разделов для администратора
- [x] выбор иконки раздела из picker
- [x] загрузка своей иконки-файла для раздела
- [x] файловая загрузка обложки раздела
- [x] hover-controls и контекстное меню раздела в дереве
- [x] создание, редактирование и удаление разделов прямо из дерева
- [x] custom confirm popup для destructive-сценариев
- [x] Inertia flow без hard refresh для KB-операций
- [x] mobile drawer / bottom-nav / header доведены до рабочего UX

## Итог этапа

Этап считаем завершенным.

Структурная часть базы знаний уже рабочая:

- разделы создаются и редактируются
- дерево и экраны живут на реальных данных
- `admin` и `employee` видят разные потоки
- экран категории по поведению уже близок к прототипу
- операции не требуют ручного `F5`
